# Guia de Estilos de Escrita

## Estilo: Interno (Slack, Notion, emails para o time)
- Tom: direto, informal mas profissional
- Sem jargão corporativo ("alavancar", "sinergia", "low-hanging fruit")
- Use bullet points para listas, evite parágrafos longos
- Seja específico com números e datas
- Exemplo: "DAU caiu 8% na última semana (Mon–Sun). Principal causa: deploy de sexta que quebrou o onboarding mobile. Bruno já tem PR de fix."

## Estilo: Externo / Clientes (release notes, emails, docs)
- Idioma: inglês
- Tom: confiante, sem ser arrogante
- Foco em valor para o usuário, não em detalhes técnicos
- Evite: "we fixed a bug" → prefira: "insights now load 2x faster"
- Exemplo: "AI Insights Feed now surfaces anomalies in real-time, so your team catches issues before users do."

## Estilo: Executivos / Board (PRDs de alto nível, updates de OKR)
- Começa com o "so what" — impacto no negócio primeiro
- Dados primeiro, narrativa depois
- Máximo 1 página / 5 bullet points por seção
- Sempre inclua: impacto esperado em $, risco principal, próximo milestone
- Exemplo: "Feature X tem potencial de reduzir churn em 0.3pp (~$180K ARR) com 6 semanas de dev."

## Estilo: Técnico / Engenharia (specs, tickets, RFCs)
- Idioma: inglês
- Seja preciso: inclua contratos de API, edge cases, critérios de aceite
- Use diagramas quando possível (descreva em texto se não tiver ferramenta)
- Inclua sempre: acceptance criteria, definição de done, dependências
