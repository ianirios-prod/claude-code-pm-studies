# Command: /retro

Analise os arquivos da sprint (reuniões, action items, métricas) e gere um relatório de retrospectiva estruturado.

## Formato de Output

# Retrospectiva — Sprint [X]
**Data:** [data]
**Time:** [participantes]

## O que Entregamos
| História | Status | Observação |
|---|---|---|

## Métricas da Sprint
- Velocity planejado vs. entregue: [X] / [Y] story points
- Action items concluídos: [X] / [Y]
- Bugs introduzidos: [X]

## O que Foi Bem
- [Item] — [por que foi bem e como repetir]

## O que Pode Melhorar
- [Problema] — [causa raiz] — [proposta de melhoria]

## Action Items para Próxima Sprint
| Responsável | Ação | Critério de Sucesso |
|---|---|---|

## Dívida Técnica Acumulada
- [Item de tech debt] — [impacto se não resolvido] — [prioridade]

---
**Regras:**
- Foco em aprendizados, não em culpas
- Cada problema deve ter uma causa raiz identificada
- Action items devem ter dono e critério de sucesso claro
- Incluir tech debt — é fácil esquecer após sprints intensos
