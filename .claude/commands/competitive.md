# Command: /competitive

Faça uma análise competitiva da feature ou produto descrito. Use web search para buscar informações atualizadas.

## Como Usar
`/competitive [feature ou produto a analisar]`

Exemplo: `/competitive "AI insights para analytics de produto"`

---

## Formato de Output

# Análise Competitiva: [Tema]
**Data:** [hoje]

## Competidores Analisados
| Competidor | URL | Tier |
|---|---|---|

## Matriz de Features
| Feature | DataFlow AI | Mixpanel | Amplitude | PostHog |
|---|---|---|---|---|

## O que Competidores Fazem Melhor
1. **[Competidor]:** [Feature/abordagem] — [por que é boa]

## Onde Temos Vantagem
1. [Nossa vantagem] — [evidência]

## Gaps que Precisamos Fechar (por prioridade)
1. **[Gap]** — mencionado por [X competidores], risco de [perda de mercado / churn]

## Oportunidades Não Exploradas pelo Mercado
- [Whitespace opportunity]

## Recomendação para o Roadmap
[O que isso implica para nossas próximas sprints]

---
**Regras:**
- Sempre buscar informações atualizadas na web (não usar só conhecimento do modelo)
- Focar em evidências: screenshots, pricing pages, release notes
- Contextualizar com nosso ICP (em `context/business-info.md`)
- Ser honesto sobre onde estamos atrás — não fazer análise enviesada
